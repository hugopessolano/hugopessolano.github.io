Esquemas de Rate Limit (rate_limit_schemas)

Defines Pydantic models and constants for managing API rate limits.

This module provides a base RateLimit model and specific subclasses (OldLimit, StorefrontLimit, BoostLimit) representing different rate limit profiles commonly encountered in APIs (e.g., Shopify). It also defines constants for burst limits and sustained rates associated with these profiles.

rate_limit_schemas.BOOST_LIMIT = 400

Burst limit for the Boost API (e.g., Shopify Plus high-volume API) rate limit (requests).

rate_limit_schemas.BOOST_RATE = 20

Sustained rate for the Boost API rate limit (requests per second).

rate_limit_schemas.BOOST_TAG = 'boost-api-rate-limit'

Identifier tag for the Boost API rate limit.

class rate_limit_schemas.RateLimit(**data)[fuente]

Bases: BaseModel

Base model representing API rate limit parameters.

Defines common attributes for controlling request frequency, concurrency, and timeouts. The delay_between_requests is calculated automatically based on the sustained_rate_rps.

Variables:

  • burst_limit (int) – The maximum number of requests allowed in a short burst. Typically corresponds to the bucket size in a token bucket algorithm.

  • sustained_rate_rps (float) – The average number of requests allowed per second over a longer period. Corresponds to the token refill rate.

  • delay_between_requests (float) – The calculated minimum delay (in seconds) required between requests to maintain the sustained rate. Automatically computed from sustained_rate_rps.

  • max_concurrent_requests (int) – The maximum number of requests that can be in flight simultaneously.

  • request_timeout (float) – The maximum time (in seconds) to wait for a response for a single API request before timing out.
