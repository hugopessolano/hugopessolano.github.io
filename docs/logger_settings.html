Configuraciones del Logger (logger_settings)

Defines settings and configurations for the Loguru logger.

This module includes: - An enumeration (LogLevel) for standard logging levels. - A function (format_assigner) to generate formatted strings for log messages. - A Pydantic model (LoggerSettings) to structure logger configurations. - Predefined instances of LoggerSettings for different logging scenarios

(e.g., stdout only, database only, both).

logger_settings.DB_LOGGER_SETTINGS = LoggerSettings(denomination='DB+OUT', level='INFO', format='<fg #808080>{time:YYYY/MM/DD HH:mm:ss}</> | <fg #FFD700>DB+OUT</> | <level>{level: <8}</level> | <cyan>{name}:{function}:{line}</cyan> - <level>{message}</level>', colorize=False, serialize=True, catch=True)

Settings for the database sink part of logs going to both DB and stdout. Requires serialization for the custom SQLite sink.

Settings for logs going only to the database. Requires serialization.

Enumeration defining standard logging levels compatible with Loguru.

Provides named constants for log severity levels.

CRITICAL = 'CRITICAL'

A serious error, indicating that the program itself may be unable to continue running.

DEBUG = 'DEBUG'

Detailed information, typically of interest only when diagnosing problems.

ERROR = 'ERROR'

Due to a more serious problem, the software has not been able to perform some function.

INFO = 'INFO'

Confirmation that things are working as expected.

SUCCESS = 'SUCCESS'

Indicates successful completion of an operation (Loguru specific).

TRACE = 'TRACE'

Detailed information, typically of interest only when diagnosing problems.

WARNING = 'WARNING'

An indication that something unexpected happened, or indicative of some problem in the near future.

class logger_settings.LoggerSettings(**data)[fuente]

Bases: BaseModel

Pydantic model representing configuration settings for a Loguru sink.

Defines the structure for logger settings, including denomination, level, format string, and flags for colorization, serialization, and exception catching.

Variables:

  • denomination (str) – An identifier for the logger/sink (e.g., «STDOUT», «DB»).

  • level (str) – The minimum log level for this sink (uses LogLevel enum values).

  • format (str | None) – The Loguru format string. If None, generated automatically.

  • colorize (bool) – Whether to apply color codes to the output (for terminals).

  • serialize (bool) – Whether to output logs as JSON (required for custom sinks like SQLiteSink).

  • catch (bool) – Whether Loguru should automatically catch exceptions in the sink.

logger_settings.format_assigner(denomination)[fuente]

Generates a Loguru format string including a custom denomination.

Constructs a format string incorporating timestamp, logger denomination, level, code location (name:function:line), and the log message itself, using Loguru’s rich text formatting tags for colorization.

Parámetros:

denomination (str) – A string identifier for the logger instance (e.g., «STDOUT», «DB»).

Devuelve:

A Loguru-compatible format string.
