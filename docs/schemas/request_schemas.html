Esquemas de Petición (request_schemas)

Defines Pydantic models for representing API requests and their prepared data.

Includes a base Request model and subclasses for specific HTTP methods (GET, POST, PUT, DELETE), handling URL construction with path parameters and query strings. Also defines RequestData to hold information needed for execution by httpx and to store/process the response.

Bases: BaseModel

Base model for representing an API request intention.

Contains the target endpoint configuration, HTTP method, and any positional arguments needed to format the final URL path.

Variables:

  • endpoint (Endpoint) – The endpoint configuration object, containing the base path template and associated metadata.

  • method (str) – The HTTP method (e.g., «GET», «POST»). To be overridden by subclasses.

  • positional_url_arguments (Optional[Tuple[str, ...]]) – A tuple of string values to substitute into the endpoint path placeholders (e.g., {arg_1}, {arg_2}).

  • endpoint_url (str) – The fully formatted URL path (without query parameters yet), generated after initialization.

model_post_init(_Request__context)[fuente]

Post-initialization hook to format the endpoint_url.

Constructs the final URL path by substituting positional_url_arguments into the placeholders ({arg_1}, {arg_2}, etc.) defined in the endpoint.path. Validates that the number of arguments provided matches the number of placeholders in the path template.

Parámetros:

  • __context (Any) – Pydantic validation context (unused).

  • _Request__context (Any)

Muestra:

ValueError – If the number of provided arguments does not match the number of placeholders in the endpoint path, or if formatting fails for other reasons.

class schemas.request_schemas.RequestData(**data)[fuente]

Bases: BaseModel

Holds prepared data for executing a single API request via httpx.

This model bridges the gap between the abstract Request models and the concrete data needed by httpx.AsyncClient. It also stores the response information after execution.

Variables:

  • url (str) – The complete, final URL for the request (including base, path, and query string).

  • headers (dict) – The dictionary of HTTP headers to send.

  • method (str) – The HTTP method (e.g., «GET», «POST»).

  • payload (Optional[Dict[str, Any]]) – The JSON payload for POST/PUT requests.

  • response_status (Optional[int]) – The HTTP status code of the received response.

  • response_headers (Optional[Dict[str, Any]]) – The headers received in the response.

  • _response (Optional[Union[Dict[str, Any], Response]]) – Private attribute storing either the raw httpx.Response object initially, or the processed JSON dictionary after the response property is accessed or set.

property response: Dict[str, Any] | None

Provides access to the processed response body as a dictionary.

If the internal _response is still an httpx.Response object (meaning it hasn’t been processed yet), this property access triggers the processing (_process_assigned_response) to extract the status code, headers, and JSON body. It then returns the processed dictionary body. If processing fails or the response was already None, returns None.

Devuelve:

The JSON response body as a dictionary,

or None if processing failed or no response exists.
