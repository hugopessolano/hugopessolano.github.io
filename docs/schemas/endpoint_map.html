Mapa de Endpoints (endpoint_map)

Defines models and functions to build a structured map of API endpoints.

This module parses a list of endpoint path strings (from endpoint_config) and organizes them into a hierarchical structure (EndpointMap) using nested EndpointGroup objects. Each endpoint path is represented by an Endpoint object, which stores the path, counts positional arguments, and links to an optional Pydantic validation model based on naming conventions.

class schemas.endpoint_map.EndpointGroup(**data)[fuente]

Bases: BaseModel

Represents a group of related endpoints, often corresponding to a resource.

Endpoints are typically categorized as “all” (listing/creating the resource collection) or “single” (accessing/modifying a specific resource instance). Nested resources are represented by child EndpointGroup objects.

Variables:

  • all (Optional[Endpoint]) – The endpoint for the collection resource (e.g., GET /products, POST /products).

  • single (Optional[Endpoint]) – The endpoint for a single resource instance (e.g., GET /products/{id}, PUT /products/{id}).

  • children (Dict[str, "EndpointGroup"]) – A dictionary mapping nested resource names to their corresponding EndpointGroup. Uses a forward reference string due to self-reference.

schemas.endpoint_map.build_endpoint_map()[fuente]

Constructs the complete, structured EndpointMap from the configuration list.

Orchestrates the process: 1. Builds the intermediate dictionary tree using build_tree_corrected. 2. Transforms the tree into nested EndpointGroup objects using tree_to_endpoint_group. 3. Validates and wraps the result in an EndpointMap (RootModel).

Devuelve:

The fully constructed and validated endpoint map object.

schemas.endpoint_map.build_tree_corrected(endpoints)[fuente]

Builds a nested dictionary representation (tree) from a flat list of endpoint paths.

Parses each path string, splitting it into segments. It organizes these segments into a hierarchical dictionary structure, distinguishing between collection endpoints (“all”) and single instance endpoints (“single”) based on path structure and the presence of trailing path parameters.

Parámetros:

endpoints (List[str]) – A list of endpoint path strings (e.g., from ENDPOINTS_LIST).

Devuelve:

A nested dictionary representing the endpoint hierarchy.

Each node has keys «all», «single», and «children».

Tipo del valor devuelto:

Dict[str, Dict]

schemas.endpoint_map.tree_to_endpoint_group(tree_node)[fuente]

Recursively transforms a dictionary tree node into an EndpointGroup object.

Converts the string paths stored in the tree (“all”, “single”) into Endpoint objects, calculating argument counts and attempting to map validation models. Recursively calls itself for child nodes.

Parámetros:

tree_node (Dict) – A dictionary node from the tree built by build_tree_corrected.

Devuelve:

The corresponding EndpointGroup object with nested children.
