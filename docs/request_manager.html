Gestor de Peticiones (request_manager)

Manages asynchronous API requests, handling authentication, rate limiting, concurrency, and pagination for Tiendanube API interactions.

Provides the RequestManager class to encapsulate API communication logic. It uses httpx for asynchronous requests, asyncio for concurrency control (Semaphore), and integrates with rate limit schemas and request schemas defined elsewhere. It also utilizes logging configured in log_db.

class request_manager.RequestManager(store_id, auth_token)[fuente]

Bases: object

Handles asynchronous execution of API requests against the Tiendanube API.

Manages authentication headers, determines and respects API rate limits, controls request concurrency using an asyncio Semaphore, executes single and multiple requests, and handles multi-page GET requests automatically.

Variables:

  • _id (str) – A unique identifier for the RequestManager instance.

  • _api_base_url (str) – The base URL for the Tiendanube API.

  • _user_agent (str) – The User-Agent string sent with requests.

  • _success_status_codes (set) – HTTP status codes considered successful.

  • _rate_limit_status_code (int) – HTTP status code indicating rate limiting.

  • _store_id (str) – The ID of the target Tiendanube store.

  • _auth_token (str) – The authentication token (prefixed with “bearer “).

  • _headers (dict) – Pre-built dictionary of request headers.

  • _composite_url (str) – The base URL combined with the store ID.

  • _rate_limit (RateLimit) – The determined rate limit profile for the store.

  • _concurrecy_semaphore (asyncio.Semaphore) – Semaphore to limit concurrent requests.

Parámetros:

  • store_id (str)

  • auth_token (str)

execute_multipage_request(request_data)[fuente]

Executes a GET request and handles pagination to retrieve all results.

Performs an initial request. If the response headers indicate multiple pages (via the “Link” header), it generates requests for the remaining pages and executes them using mass_execute_requests.

Parámetros:

request_data (Request) – The initial GET request schema object. Must be a GET request.

Devuelve:

A list containing the RequestData objects for

all pages, including the initial request. Returns only the first request’s result if pagination is not applicable or fails.

Tipo del valor devuelto:

List[RequestData]

Muestra:

ValueError – If the input request_data is not a GET request.

async execute_requests(request_data_list)[fuente]

Executes a list of prepared requests concurrently, respecting rate limits.

Manages concurrency using the semaphore and implements delays between requests during the sustained phase based on the calculated rate limit. Uses rich.progress.track for visual feedback.

Parámetros:

request_data_list (List[RequestData]) – A list of prepared RequestData objects.

Devuelve:

A list containing the RequestData objects, each

updated with its corresponding httpx.Response (or None if failed). The order matches the input list.
